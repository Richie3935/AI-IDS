# AI-Powered Intrusion Detection System (IDS) — Version 7

A modular Python IDS built with Scapy. The system includes live packet
capture, real-time traffic statistics, rule-based detection, network flow
generation, **ML-based classification (v7)**, persistent MySQL alert storage,
and a Flask monitoring dashboard.

## Features

- Live packet capture using Scapy
- Thread-safe traffic statistics with periodic JSON snapshots
- Rule-based detection for port scans, TCP SYN floods, and ICMP floods
- Network flow generation keyed by 5-tuple (src IP, dst IP, src port, dst port, protocol)
- Automatic inactive-flow timeout and bounded completed-flow storage
- CSV export for completed flow records
- **[v7] Random Forest ML classifier trained on CICIDS2017**
- **[v7] Real-time flow classification with per-alert confidence scores**
- **[v7] Hybrid detection pipeline: Rule Engine → ML Engine → Alert Manager**
- MySQL alert persistence with connection pooling and parameterised queries
- Graceful handling of database unavailability
- Flask dashboard with Bootstrap 5 and Chart.js
- Searchable alert history with attack type and severity filters
- **[v7] Dedicated ML Alerts page showing confidence scores**
- **[v7] Rule vs ML alert breakdown chart on the Statistics page**

## Requirements

- Python 3.11+
- MySQL Server 8.0+ or compatible instance
- Administrator/root privileges for live packet capture
- Npcap on Windows: <https://npcap.com/>

Install all Python dependencies:

```bash
pip install -r requirements.txt
```

## MySQL Setup

Create the database and alerts table (v7 schema includes `confidence` and
`ml_generated` columns):

```bash
mysql -u root -p < database/schema.sql
```

**Existing deployment migration.** If you already have the v6 schema, run the
`ALTER TABLE` block printed as a comment at the bottom of `database/schema.sql`
against your live database to add the two new columns without data loss.

Configure the connection with environment variables (use `export` on Linux/macOS):

```bash
set AI_IDS_DB_HOST=localhost
set AI_IDS_DB_PORT=3306
set AI_IDS_DB_USER=root
set AI_IDS_DB_PASSWORD=your_password
set AI_IDS_DB_NAME=ai_ids
set AI_IDS_DB_POOL_SIZE=5
```

## Model Training (Version 7)

Place the CICIDS2017 MachineLearningCSV files in the `datasets/` directory,
then run the training pipeline:

```bash
python ml/train_model.py
```

The script will:

1. Load and concatenate all CSV files from `datasets/`
2. Filter to three target classes: `BENIGN`, `PortScan`, `DoS`
3. Extract five features matching the live FlowGenerator output
4. Train a Random Forest with 5-fold stratified cross-validation
5. Print accuracy, precision, recall, F1, confusion matrix, and CV scores
6. Save the model bundle to `ml/model.pkl`

Optional training flags:

```bash
python ml/train_model.py --n-estimators 200 --max-depth 20
python ml/train_model.py --dataset-dir datasets/ --output ml/model.pkl
python ml/train_model.py --cv-folds 10 --log-level DEBUG
```

## Running the IDS

Basic capture (requires root/Administrator):

```bash
sudo python main.py
```

Specific interface and BPF filter:

```bash
sudo python main.py -i eth0 -f "tcp"
```

Stop after a fixed packet count:

```bash
sudo python main.py -c 500
```

Disable per-packet console output:

```bash
sudo python main.py --no-display
```

### ML Engine Options (Version 7)

```bash
# Use a custom model path
sudo python main.py --model-path ml/model.pkl

# Raise the confidence threshold (fewer but higher-confidence ML alerts)
sudo python main.py --ml-confidence-threshold 0.80

# Disable the ML layer and run rule-based detection only
sudo python main.py --disable-ml
```

### Rule Thresholds

```bash
sudo python main.py --port-scan-window 30 --port-scan-ports 8
sudo python main.py --syn-window 5 --syn-threshold 50
sudo python main.py --icmp-window 5 --icmp-threshold 25
sudo python main.py --disable-rules
```

### Flow Generation Options

```bash
sudo python main.py --flow-timeout 60 --flow-cleanup-interval 10
sudo python main.py --flow-csv ml/completed_flows.csv
sudo python main.py --disable-flow-export
```

Completed flows are exported with these ML-ready columns:
`src_ip`, `dst_ip`, `src_port`, `dst_port`, `protocol`,
`packet_count`, `byte_count`, `flow_duration`,
`packets_per_second`, `bytes_per_second`, `start_time`, `end_time`

## Hybrid Detection Architecture

```text
Network Traffic
      │
      ▼
PacketCapture (Scapy)
      │
      ├──▶ TrafficStats      periodic JSON snapshot for dashboard
      │
      ├──▶ RuleEngine        port scan / SYN flood / ICMP flood
      │         │
      │         └──▶ MySQLAlertRepository   (ml_generated = 0)
      │
      └──▶ FlowGenerator     5-tuple flow aggregation
                │
                └──▶ AIEngine (Random Forest, every stats interval)
                          │
                          └──▶ MySQLAlertRepository  (ml_generated = 1,
                                                       confidence = 0.xx)
                                      │
                                      └──▶ Flask Dashboard
```

The Rule Engine fires immediately on individual packets. The ML Engine
classifies completed flows at every stats report interval (default: 10 s),
so it catches patterns that span multiple packets but are missed by
per-packet signatures.

## Alert Storage (Version 7 Schema)

The `alerts` table stores:

| Column | Type | Notes |
|--------|------|-------|
| `id` | `BIGINT` | Auto-increment primary key |
| `timestamp` | `DATETIME` | Alert creation time |
| `source_ip` | `VARCHAR(45)` | IPv4 or IPv6 source |
| `destination_ip` | `VARCHAR(45)` | Nullable |
| `attack_type` | `VARCHAR(100)` | Rule label or ML class |
| `severity` | `VARCHAR(30)` | CRITICAL / HIGH / MEDIUM / INFO |
| `description` | `TEXT` | Human-readable summary |
| `confidence` | `DECIMAL(5,4)` | ML probability 0.0–1.0, NULL for rule alerts **(v7)** |
| `ml_generated` | `TINYINT(1)` | 1 = ML engine, 0 = rule engine **(v7)** |

## Cybersecurity Notes

- Run packet capture only on networks and systems where you have authorisation.
- Use least privilege except where elevated capture permissions are required.
- Protect logs and database records — they may contain sensitive IP metadata.
- Treat the rule engine and ML model as baseline detection layers, not replacements for full enterprise monitoring.
- The ML model was trained on CICIDS2017 (2017 traffic). Retrain periodically on recent traffic for best performance.
- Review low-confidence ML alerts carefully before taking automated action.
